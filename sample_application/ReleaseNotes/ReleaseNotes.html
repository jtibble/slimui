Release Notes

In Development

  • Removing controllers completely and refactoring dependency-injection.
  • Refactoring config.json paths.
  • Updating examples to show model reuse across screens.

v0.0.13 Stable April 16, 2014

  • Removed RequireJS script-loading due to work done with Gulp. Now the framework assumes that all scripts are loaded in-memory before application is initialized

v0.0.12 April 8, 2014

  • Bug Fix: Setting ControllerCommunication channel to value [false] would not notify channel-listeners
  • Updated Documentation for ngSantitize
  • Setup Information added
  • Documentation for Services updated

v0.0.11 April 7, 2014

  • Added ngSanitize to allow for rendering of HTML in a template

v0.0.10 April 4, 2014

  • Added ability to pass parameters to HTML elements
  • Fixed bug when defaultView requires parameters

v0.0.9 March 27, 2014

  • Fixed parameter-passing bug. Now passing parameters using Router.goTo("state", {'parameterName': value}) works correctly.
  • Improved config-file validation and error-checking to give more helpful advice.

v0.0.8 March 25, 2014

  • Added Services Examples to help developers write services.
  • Refactored config.json service-generation functionality to act more like views/components.
  • Added arbitrarily-deep subroutes with Parameter Passing. Additionally, extended examples.

v0.0.7 March 21, 2014

  • Rebuilt Parameter Passing functionality with examples
  • Created this Release Notes page
  • Config.json: Restructured and simplified. Added assumptions about file names and naming conventions to automate loading files, which greatly reduces config.json complexity. Added support for multiple root-folders, specified on a view-by-view basis. Updated config.js examples and documentation.

v0.0.6 March 20, 2014

  • Refactored framework to separate Model, Modelbuilder, Actions, and Controller. See Simple Page Documentation.
  • Reworked route-generation to allow for view to have a url of '/'
  • In config.json, renamed 'regions' to 'subroutes'

v0.0.5 March 14, 2014

  • Changed 'FrameworkPOST' service to 'FrameworkAJAX'
  • Removed 'Session' service
  • Removed two 'Routing' service methods
